MeetBrief / Setup

Up and running
in under 5 minutes

MeetBrief is an Electron app. If you have Node.js and an API key, you're ready. No installers, no accounts, no signup — just clone and run.

🍎 macOS 🪟 Windows 🐧 Linux
Step 1

Prerequisites

⬢
Node.js 18 or newer Download from nodejs.org or install via nvm. Run node --version to check.
📦
npm (comes with Node) No separate installation needed — npm is bundled with every Node.js release.
🔑
API key — pick your AI provider Claude (Anthropic) · OpenAI GPT-4o · Gemini (Google AI Studio) · or install Ollama for fully offline use (no key needed).
🍎
macOS: Screen Recording permission System audio capture requires Screen Recording access. On first launch, MeetBrief will prompt for it automatically via System Preferences → Privacy & Security.
Step 2

Clone and install

MeetBrief lives in the MeetingCopilot/ directory of the TinyTools repository.

1

Clone the repository

The MeetingCopilot/ subfolder contains the full Electron app. You only need this subfolder.

2

Enter the directory

Navigate into MeetingCopilot/ — this is where package.json and the Electron entry point live.

3

Install dependencies

npm will fetch Electron, @huggingface/transformers, and other runtime dependencies.

terminal 
# Clone the full repo
git clone https://github.com/martian-coder/TinyTools.git

# Enter the app directory
cd TinyTools/MeetingCopilot

# Install all dependencies
npm install
Step 3

Choose your AI provider

MeetBrief supports four providers. Pick the one that fits your workflow — all configuration is done inside the app in Settings. Your key stays on your machine and is never sent to MeetBrief.

Recommended. Get your API key at console.anthropic.com → API Keys. Paste it in Settings → Claude API. Billing is per-token directly with Anthropic — typically under $0.10/hour with prompt caching on.
OpenAI GPT-4o. Get your API key at platform.openai.com → API Keys. Paste it in Settings → OpenAI. Default model is gpt-4o-mini (fast and affordable); switch to gpt-4o for higher quality.
Gemini Live API. Get a key at aistudio.google.com. Paste it in Settings → Gemini BYOK. No audio driver setup needed on macOS — Gemini handles audio streaming natively via ScreenCaptureKit.
Fully offline. Install Ollama from ollama.com, run ollama pull llama3.2 (or any model), then set the Ollama host and model name in Settings → Local AI. No API key needed. Latency depends on your GPU.
Your API key stays on your machine. Keys are stored locally in the app's data directory and sent directly to your chosen AI provider. They never touch MeetBrief infrastructure — there is none.

Step 4

Launch MeetBrief

Once dependencies are installed and .env is configured, you're ready to start.

1

First run — Whisper download

On the very first launch, MeetBrief will download the Whisper model (~250MB for small) and cache it locally. This is a one-time operation.

2

Grant permissions (macOS)

macOS will prompt for Screen Recording permission. Grant it — this is required for system audio capture to work. You can verify it in System Settings → Privacy & Security → Screen Recording.

3

Load context, start session

Paste your meeting context into the text area, choose your provider, and click Start Session. Join your meeting — MeetBrief will begin listening.

terminal 
# Start MeetBrief
npm start

# The app window will open.
# On first run, Whisper downloads (~250MB).
# Accept the Screen Recording prompt on macOS.

Troubleshooting

Common issues

No system audio is being captured
macOS: Check System Settings → Privacy & Security → Screen Recording and ensure MeetBrief is listed and enabled. You may need to restart the app after granting permission. Windows: Ensure the default playback device is set correctly. Linux: System audio requires a PulseAudio loopback module. Run pactl load-module module-loopback to test.
API calls fail with an authentication error
Open Settings inside MeetBrief and re-paste your API key. For Claude keys start with sk-ant-; for OpenAI keys start with sk-. If the error persists, check that you have credits/billing enabled on your provider's dashboard.
Whisper download fails or hangs
The model downloads from Hugging Face CDN (~250MB for whisper-small). If it fails, check your internet connection and try again. The model is cached in ~/.cache/meetbrief/ after the first successful download. Switching to WHISPER_MODEL=Xenova/whisper-tiny in your .env downloads a smaller model (~75MB).
Ollama not connecting
Ensure Ollama is running (ollama serve) and accessible at http://localhost:11434. Verify your model is pulled: ollama list. The model name in OLLAMA_MODEL must exactly match the name shown by ollama list.
